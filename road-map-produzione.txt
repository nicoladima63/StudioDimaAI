ROADMAP DEPLOYMENT PRODUZIONE - Step by Step

  1. Preparazione Codice
  - Build React: npm run build in /client
  - Test che tutto funzioni in dev
  - Commit finale su git

  2. Sul Server (Windows/Linux)
  - Installa Python 3.9+
  - Installa Node.js
  - Clona repository: git clone [url]

  3. Setup Backend
  - cd server
  - python -m venv venv
  - venv\Scripts\activate (Windows) / source venv/bin/activate (Linux)
  - pip install -r requirements.txt
  - pip install gunicorn (se Linux) / pip install waitress (se Windows)

  4. Configurazione Ambiente
  - Copia .env.example → .env
  - Configura variabili produzione (DB paths, API keys, etc.)
  - Setta modalità prod: crea server/instance/database_mode.txt con prod

  5. Setup Frontend
  - cd client
  - npm install
  - npm run build
  - Copia cartella dist/ in /server/static/

  6. Test Locale
  - Avvia: python -m server.app.run
  - Verifica tutto funzioni

  7. Servizio Produzione
  - Windows: Crea servizio Windows o usa Task Scheduler
  - Linux: Crea systemd service
  - Script avvio con Gunicorn/Waitress

  8. Nginx (Opzionale ma consigliato)
  - Installa Nginx
  - Configura reverse proxy
  - Setup SSL se serve

  9. Monitoraggio
  - Log rotation
  - Backup automatico
  - Health check

  10. Go Live
  - Start servizio
  - Test finale
  - Monitor primi giorni


  Ho creato 3 script automatizzati:

  1. deploy.bat - Script principale che:
  - Verifica connessione SERVERDIMA (192.168.1.200)
  - Crea backup automatico installazione precedente
  - Build React frontend
  - Copia tutto sul server

  - Crea script avvio

  2. check_deployment.bat - Verifica deployment
  - Controlla tutti i file necessari
  - Test connessione server
  - Mostra status configurazione

  3. requirements-prod.txt - Dipendenze produzione aggiuntive

  Come usare:
  1. Esegui deploy.bat dal tuo PC
  2. Esegui check_deployment.bat per verificare
  3. Sul server vai in \\SERVERDIMA\StudioDimaAI\ ed esegui start_server.bat
  4. Apri http://192.168.1.200:5000

  Lo script gestisce tutti gli errori comuni e ti dice esattamente cosa fare se qualcosa va storto.

  Perfetto! Ho creato 2 script per gestire l'automazione:

  1. start_automation.bat - Configura tutti i servizi automatici:
  - Crea automation_service.bat sul server per avvio manuale
  - Crea automation_task.xml per Task Scheduler Windows (avvio automatico)
  - Gestisce tutti i processi in background già presenti nel tuo scheduler.py:
    - Sincronizzazione calendari automatica
    - SMS promemoria appuntamenti
    - SMS richiami pazienti

  2. check_automation.bat - Verifica stato automazione:
  - Controlla file necessari
  - Mostra log recenti
  - Verifica processi attivi
  - Status Task Scheduler

  Come funziona:
  1. Il tuo scheduler.py è già completo con tutti i job automatici
  2. Gli script creano solo l'infrastruttura per eseguirlo in produzione
  3. L'automazione gira in background separata dal server web
  4. Tutto configurabile dal pannello web (Impostazioni > Automazione)

  Sequenza deployment completa:
  1. deploy.bat - Installa tutto
  2. start_server.bat - Avvia server web (sul server)
  3. start_automation.bat - Configura automazione (dal tuo PC)
  4. automation_service.bat - Avvia automazione (sul server)
