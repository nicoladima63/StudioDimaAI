# ЁЯУМ Progetto: Automazione Registrazione Smaltimento Rifiuti - RENTRI

## ЁЯОп Obiettivo

Costruire un'applicazione Python che automatizzi la registrazione dei movimenti di smaltimento rifiuti per uno studio odontoiatrico, a partire dalla ricezione PEC del FIR, fino all'inserimento nel portale RENTRI.

---

## ЁЯз▒ Stack Tecnologico

* Python 3.10+
* Librerie: `imaplib`, `email`, `pdfplumber`/`PyMuPDF`, `requests`, `dotenv`, `schedule`, `openpyxl`, `pandas`
* Storage: File system locale + opzionale SQLite o Excel
* Sicurezza: Gestione token OAuth2 (RENTRI)
* IDE: Visual Studio Code

## ЁЯФД Fasi del Progetto

### 1. Configurazione ambiente

#### ЁЯФС Ottenere le credenziali API RENTRI

Per ottenere `client_id` e `client_secret` da usare con le API RENTRI:

1. Vai su [https://www.rentri.gov.it](https://www.rentri.gov.it)
2. Accedi con SPID o CNS come titolare dello studio
3. Entra nella sezione "Accesso API" o "Credenziali applicative"
4. Richiedi una nuova applicazione (es. `automazione-smaltimento`)
5. Salva i valori generati e aggiungili nel file `.env`:

   ```
   RENTRI_CLIENT_ID=tuo_client_id
   RENTRI_CLIENT_SECRET=tuo_client_secret
   RENTRI_TOKEN_URL=https://api.rentri.gov.it/oauth2/token
   ```

* Creare file `.env` con:

  * Credenziali PEC (email, password)
  * Parametri IMAP server
  * Client ID e Secret RENTRI
* Setup struttura progetto:

  * `src/`

    * `pec_reader.py`
    * `rentri_client.py`
    * `parser.py`
    * `logger.py`
    * `main.py`
  * `data/` per i PDF scaricati
  * `logs/` per log automatici
  * `output/` per Excel o DB

---

### 2. Lettura PEC

* Connettersi via IMAP
* Cercare PEC con oggetto contenente "FIR" o mittente noto
* Scaricare allegato PDF nella cartella `data/`
* Rinominare con formato: `YYYYMMDD_FIR_NUMERO.pdf`
* Loggare evento

### 3. Parsing del PDF

* Estrarre:

  * Numero FIR
  * Data
  * Codice CER
  * Quantit├а in kg
  * Trasportatore / Smaltitore
* Salvare i dati in un dizionario Python
* Opzionale: Validare firma digitale (futuro)

---

### 4. Autenticazione RENTRI (OAuth2 / JWT con CNS o certificato dominio)

> Esistono due modalit├а principali per autenticarsi con le API RENTRI:
>
> тЬЕ Modalit├а semplificata (Client ID + Secret)
> ЁЯФР Modalit├а avanzata (firma con certificato CNS o P12)

#### тЬЕ Metodo 1 тАУ OAuth2 con `client_id` e `client_secret`

* Endpoint: `POST /oauth2/token`
* Flow: `grant_type=client_credentials`
* Usare token ricevuto come Bearer nei successivi header
* Salvare token temporaneamente in file `.json`
* Funzione di refresh token automatico se 401 Unauthorized

#### ЁЯФР Metodo 2 тАУ JWT firmato con CNS / certificato P12

Richiede:

* Lettura del certificato CNS installato oppure file `.p12`
* Firma digitale del JWT con chiave privata
* Digest del corpo della richiesta
* Header `Authorization`, `Digest`, `Agid-JWT-Signature`

**Header richiesti:**

```http
Authorization: Bearer <JWT>
Digest: SHA-256=<digest base64>
Agid-JWT-Signature: <JWT firmato per integrit├а>
```

**Esempio base flusso:**

1. Costruzione del JWT (con jti, issuer, audience)
2. Firma con RSA o ECDSA in base al certificato
3. Calcolo SHA256 del corpo della richiesta
4. Costruzione secondo token `idAuth` e `integrity`
5. Invio della richiesta con i 3 header

*(Vedi esempi dettagliati su [https://api.rentri.gov.it/docs?page=esempi](https://api.rentri.gov.it/docs?page=esempi))*

* Endpoint: `POST /oauth2/token`
* Ottenere token con `client_id` e `client_secret`
* Salvare token temporaneamente in file `.json`
* Funzione di refresh token automatico se 401 Unauthorized

### 5. Verifica o creazione registro attivo

* Endpoint: `GET /registri` тЖТ controlla se c'├и registro attivo
* Se non esiste, crealo (opzionale)

### 6. Inserimento movimento di scarico

* Endpoint: `POST /registri/{id}/movimenti`
* Usare i dati estratti dal PDF
* Inviare richiesta JSON con i dati smaltimento
* Verificare risposta e loggare esito

### 7. Gestione degli errori

* Gestione automatica errori:

  * Token scaduto тЖТ refresh
  * Mancanza dati PDF тЖТ log warning
  * Errori API тЖТ retry/backoff o alert

### 8. Reportistica e archivio

* Inviare email periodica o notifiche (opzionale)

### 9. Scheduler
